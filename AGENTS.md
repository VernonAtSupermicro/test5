# 專案 Agent 規定

本文件為本專案對所有 Cursor Agent 的持久化規定。之後開新 agent 時無需使用者再次說明；請一律遵守。

## 專案性質

- 這是**靜態網頁**專案（HTML / CSS / JavaScript）。
- 必須可在**手機、平板、電腦**上直接開啟執行（不依賴必須建置的後端或複雜工具鏈）。
- 介面文字**全部使用繁體中文**。
- 必須具備 **RWD**：直式與橫式螢幕都要正常運作、可讀、可操作。

## 語言與溝通

- 與使用者對話一律使用**繁體中文**（含回覆、說明、commit message 說明、語音相關內容的處理）。
- 程式碼註解、UI 文案、錯誤訊息、README 說明優先使用繁體中文。

## Git 工作流程（強制）

- 每次改完檔案後，**直接 commit 並 push 到 GitHub 的 `main` branch**。
- **不要**另外開 feature branch；**不要**改走 PR 流程（除非使用者當次明確要求）。
- 自動產生清楚、描述性的 **commit message**（繁體中文或中英清楚皆可，但要能看出改了什麼）。
- 完成一輪變更後的標準步驟：`git add` → `git commit` → `git push origin main`。

## 實作原則

- 優先純靜態、可直接用瀏覽器開啟的結構。
- 版面需同時顧及直向與橫向、小螢幕與大螢幕。
- 不擅自加入與需求無關的相依套件或建置流程。
